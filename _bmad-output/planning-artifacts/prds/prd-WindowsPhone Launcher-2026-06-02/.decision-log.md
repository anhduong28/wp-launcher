# Decision Log: WindowsPhone Launcher PRD

## 2026-06-02

- Created PRD workspace.
- Product concept: Android launcher inspired by Windows Phone live tiles, centered on a dynamic tile-based home screen, tile customization, glanceable information, and a minimalist experience.
- Upstream product brief exists but is still draft/skeleton: `_bmad-output/planning-artifacts/briefs/brief-WindowsPhone Launcher-2026-06-02/brief.md`.
- Document output language: English.
- Conversation language: Vietnamese.
- Discovery research captured in `addendum.md`; comparable launcher space is active, so differentiation must be explicit in the PRD.
- Working mode selected: Coaching path.
- Entry point selected by facilitator: Journey-led, because the launcher is UX-heavy and consumer-facing.
- Captured UJ-1: former Windows Phone 8.1 user starts from a smooth, colorful, animated Live Tile grid on Android.
- Captured glossary terms: Launcher, Tile, Live Tile, Tile Grid, Tile Size.
- Product direction includes all three Live Tile levels: notification-derived status, first-class core Tiles, and Windows Phone 8.1-like animation behavior.
- Scope caution: MVP still needs a narrow first slice to protect smoothness and delivery feasibility.
- MVP direction selected: Authentic Start Screen MVP plus useful live information, not decorative tiles only.
- First-class Live Tile priority: Weather, Clock, SMS, Phone, Calendar.
- MVP Tile sizes selected: `1x1`, `2x2`, and `4x2`. `4x4` is out of scope for MVP.
- Start Screen MVP interactions approved: pin app, unpin app, resize Tile, rearrange Tile, tap Tile to launch app, long press Tile to enter edit mode.
- App browsing surface selected: Windows Phone-style vertical alphabetical App List with search. Traditional Android grid app drawer is not the MVP default.
- App List navigation selected: right-to-left horizontal swipe from Start Screen opens App List.
- App List return navigation selected: left-to-right horizontal swipe from App List returns to Start Screen.
- Android Back from App List returns to Start Screen.
- Tile color model selected: default vivid multi-color Tiles plus optional unified accent color mode.
- Start Screen background selected: classic black by default, with optional user-selected image background behind Tiles.
- MVP Live Tile animation selected: Windows Phone-inspired flip or face-switching behavior.
- Weather Tile data source requirement: use a free weather data source; exact provider remains open for technical research.
- Weather Tile location strategy: manual city/location entry by default, optional auto-location only after user opt-in.
- SMS and Phone Tile content strategy: show counts only in MVP, no message preview or caller preview.
- Calendar Tile content strategy: show today's event count only in MVP, no event title/location/attendee preview.
- Clock Tile content strategy: digital clock in MVP; analog style out of scope.
- MVP settings approved: color mode, accent color picker, black/image background, manual weather location with optional auto-location, and Live Tile animation on/off.
- MVP primary success criteria: users can use it as primary launcher for 7 days without heaviness/lag complaints; Start Screen subjectively feels like Windows Phone 8.1.
- Project positioning selected: public free app.
- PRD cleanup: removed resolved open questions about tile sizes, MVP direction, and first-class tile categories.
