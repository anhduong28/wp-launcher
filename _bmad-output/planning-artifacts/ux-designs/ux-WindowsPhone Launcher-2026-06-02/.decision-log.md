# Decision Log: WindowsPhone Launcher UX

## 2026-06-02

- Created UX workspace.
- Conversation language: Vietnamese.
- Document output language: English.
- Intended outputs: `DESIGN.md` for visual identity and `EXPERIENCE.md` for information architecture, behavior, states, interactions, accessibility, and key flows.
- Working mode selected: Coaching path.
- Proposed source inputs: PRD and technical research as primary UX inputs; product brief only as secondary context because it remains skeletal.
- UX foundation: Android phone portrait-first for MVP. Landscape and tablet support are deferred.
- Start Screen layout: 4-column vertical Tile Grid. `1x1` spans 1 column, `2x2` spans 2 columns, `4x2` spans all 4 columns.
- Tile spacing: tight spacing to preserve Windows Phone 8.1 Start Screen feel. Spacing customization is out of MVP.
- Start Screen chrome: minimal Windows Phone-style layout with no dedicated header/status/search/settings area above the Tile Grid in MVP.
- Settings entry: launcher-owned Settings appears as an app/tile entry that can be pinned to the Start Screen.
- Edit Mode: Windows Phone-style direct tile controls. Long press makes the Tile feel lifted/active and shows small corner controls for unpin and resize. Bottom sheet actions are out of MVP.
- Resize interaction: user resizes Tiles by dragging a Tile corner in Edit Mode. This requires prototype validation for grid stability and mobile ergonomics.
- Resize layout behavior: if the target Tile size needs more space, the grid reflows and pushes affected Tiles downward instead of blocking the resize.
- Rearrangement interaction: in Edit Mode, user drags a Tile freely and releases it to snap into the 4-column grid; affected Tiles reflow around the new position.
- Live Tile motion cadence: supported Tiles flip on a light 8-12 second interval per Tile, phase-offset so Tiles do not animate all at once.
- Live Tile animation off behavior: Tiles enter static app-tile mode; flip animation and live information updates are both disabled for affected Tiles.
- Tile content hierarchy: Weather and Clock Tiles are information-first; ordinary app Tiles are icon-first with count/status as secondary content.
- Weather Tile setup state: before location is configured, the Tile shows a setup state with "Set location"; tapping it opens Weather setup.
- Permission-missing Tile state for SMS, Phone, and Calendar: show normal app Tile/icon with no count. Do not show permission prompts or "Enable" badges inside these Tiles in MVP.
- Permission education: first-run onboarding introduces optional Live Tile permissions. Each permission remains explicit opt-in according to Android platform rules.
- Onboarding MVP: three steps - Welcome, Set as default launcher, Live Tile permissions.
- Onboarding Live Tile permissions: offer optional Notification access, Calendar access, and Location access. Direct SMS/Phone permissions are not requested in MVP onboarding.
- App List visual: vertical lowercase-style Windows Phone list with app icon, app name, and alphabet headers. Alphabet jump list is deferred.
- App List search: fixed search field at the top of the App List.
- Typography direction: more modern Android readability, with clearer labels and slightly heavier font weight than original Windows Phone, while staying minimal.
- Tile shape: sharp square corners like Windows Phone; no rounded corners in MVP.
- Default Tile palette: Windows Phone classic colors - cyan, blue, green, red, purple, orange.
- Tile icon direction: use white/monochrome icons when feasible to evoke Windows Phone. Fall back to original app icon when safe monochrome conversion is not possible.
- Image background behavior: Tiles remain solid color; the image appears only behind the Tile Grid/in gaps, preserving Tile contrast.
- Reduced motion behavior: MVP follows only the in-app Live Tile animation setting and does not automatically follow Android system reduced/remove animation settings. Accessibility review should revisit this.
- Default first-run Start Screen Tiles after onboarding: Weather `4x2`, Clock `2x2`, SMS `2x2`, Phone `1x1`, Calendar `2x2`, Settings `1x1`.
- Validation ran with all lenses: rubric walker, accessibility, technical feasibility, and product coherence.
- Validation revision: onboarding no longer asks for Location generally; Location is deferred to Weather Setup as optional auto-location.
- Validation revision: Live Tile animation off disables flip/face-switch motion only; supported live data remains visible as a non-animated single-face state where available, aligning UX with PRD FR-25.
- Validation revision: Android system remove-animation / animator-duration accessibility settings should stop Live Tile flip motion, while the in-app toggle remains the explicit product control.
- Validation revision: added 48dp hit target requirements, TalkBack/custom action requirements, contrast-safe foreground switching, default-launcher fallback states, notification-access system-settings route, provider-neutral Weather states, and Tile Grid prototype gate.
- Weather provider decision: Open-Meteo is the default MVP provider. Manual Weather setup uses city/location search with resolved selectable result; optional auto-location remains secondary. Public-release implementation must verify current Open-Meteo quota, attribution, privacy, and commercial-use terms.
